# Cryptura Payment Integration

Этот репозиторий содержит пример современной интеграции с платежным шлюзом **Cryptura**.  
Мы перешли от громоздких подписей и фиатных методов к гибкому крипто-инвойсингу.

## Документация API

Актуальные эндпоинты и спецификации:

- **Base URL:** `https://api.cryptura.io`
- **Docs:** Cryptura Gitbook

## Разбор компонентов (`assas.py`)

### 1) `RequiredMethodsMixin`

**Зачем это нужно:** это наш «страховой полис».  
Миксин проверяет, чтобы любой новый контекст или провайдер реализовал все необходимые методы с правильными аргументами ещё на этапе инициализации.

Если вы забудете про `parse_payin_response`, код упадёт с понятной ошибкой, а не во время живого платежа.

### 2) `CrypturaPaymentContext`

**Назначение:** контейнер для данных платежа.

**Поля:**
- сумма;
- ID заказа;
- валюта (`USDT`, `BTC` и т.д.);
- URL-адреса для редиректов.

**Методы:**
- `get_payin_payload` — формирует чистый JSON без лишнего мусора, который ожидает увидеть Cryptura;
- `parse_payin_response` — ищет в ответе сервера `checkout_url` (ссылка, на которую отправляем клиента для оплаты).

### 3) `BaseCrypturaProvider`

**Назначение:** сердце интеграции.

- **Auth:** забудьте про MD5-хеширование. Теперь используется стандартный `Bearer Token` в заголовках — быстрее, чище и безопаснее.
- `check_empty_response`: проверяет статус ответа. Если Cryptura сообщает об ошибке, мы бросаем `RequestException` с деталями.

### 4) `Provider` (конечный класс)

Это то, с чем работает основной код (бизнес-логика).

- Автоматически подтягивает `CRYPTURA_API_KEY` из настроек.
- Метод `get_requisites` возвращает всё необходимое для запуска оплаты.

---

## Поток данных (Data Flow)

1. **Вызов:** `Provider().get_requisites(amount, order_id)`
2. **Сборка:** создаётся `CrypturaPaymentContext`
3. **Запрос:** провайдер отправляет `POST` на `/v1/invoices/create` с Bearer-токеном
4. **Валидация:** проверка статус-кода и JSON-ответа
5. **Результат:** вы получаете `pay_url` и отправляете пользователя по этой ссылке — остальное Cryptura берёт на себя

---

## Настройка (Django Settings)

Для работы интеграции добавьте в `settings.py`:

```python
# API ключ из личного кабинета Cryptura
CRYPTURA_API_KEY = "your_secret_token_here"

# Базовый URL вашего сайта для формирования callback-ссылок
BASE_SITE_URL = "https://your-domain.com"
```

> **Важно:** Cryptura работает по протоколу **HTTPS**.  
> Убедитесь, что `BASE_SITE_URL` использует защищённое соединение, иначе callback-уведомления (вебхуки) могут не дойти.
